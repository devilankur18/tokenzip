# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | ✅ Yes             |
| < 1.0.0 | ❌ No              |

## Reporting a Vulnerability

We take the security of TokenZip seriously. If you believe you have found a security vulnerability, please report it to us as follows:

1.  **Do not open a GitHub issue** for security vulnerabilities.
2.  Send an email to **ankur@example.com** with the details of the vulnerability.
3.  We will acknowledge your report within 48 hours.
4.  We will provide a timeline for a fix and keep you updated on the progress.

Please include the following in your report:
- Type of issue (e.g., buffer overflow, SQL injection, cross-site scripting, etc.)
- Full paths of source file(s) related to the manifestation of the issue
- The location of the affected code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit the issue

Thank you for helping keep TokenZip secure!
